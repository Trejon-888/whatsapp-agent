How It Works

Think of it like a neural network. Input comes in, the agent processes it with context, output goes back. Every message follows this exact cycle:

💬Input
→
📡Receive
→
📖Context
→
🧠Think
→
✅Output

Example: Simple Question

INPUT Customer texts WhatsApp
"Do you guys deliver?"
CONTEXT Bridge loads BUSINESS.md + conversation history
Delivery: within 5 miles, $5 fee, minimum $20
Hours: Mon-Thu 12-9, Fri-Sat 12-10, Sun 1-8
Previous messages: first contact, no history
THINK claude -p processes the prompt
Pipe mode: reads business context + question, generates reply
OUTPUT Reply sent to WhatsApp
"Yes! We deliver within 5 miles of the shop. $5 delivery fee, $20 minimum order. Just send us what you'd like and your address!"

Example: Follow-up with Memory

INPUT Same customer, 2 minutes later
"Cool, can I get 2 scoops of mango and a chocovolcano?"
CONTEXT Bridge loads BUSINESS.md + previous messages
Menu: Double scoop $6.50, Chocovolcano sundae $9
Mango Sorbet: available (dairy-free)
Previous: asked about delivery, told 5 miles/$5/$20 min
THINK claude -p has full conversation context
Knows they want delivery. Calculates total. Asks for address.
OUTPUT
"Great choices! That's $6.50 for the double mango scoop + $9 for the Chocovolcano + $5 delivery = $20.50 total. What's your delivery address?"

Example: Escalation

INPUT
"I want to book your ice cream cart for a wedding, 200 guests, June 15. Can you do a custom menu?"
CONTEXT
Catering: starting $250, up to 30 guests, 3 flavors
200 guests = way beyond standard package
Custom menu = not in FAQ, needs owner
ESCALATE Agent flags it for the business owner
Reason: Custom catering for 200 guests, beyond standard package
Logged to: escalations/2026-03-30.md
OUTPUT Customer still gets a warm reply
"A wedding with 200 guests sounds amazing! That's a custom job so I want to make sure we get it perfect. Let me have Enrique reach out to you directly to plan the menu. He'll be in touch shortly!"
The key insight: Each message goes through the same cycle. Input arrives, the bridge loads your business context + conversation history, pipes it to Claude, and sends the output. The "training" is just a markdown file. Edit BUSINESS.md and the agent instantly knows new things. No redeployment, no retraining.

LEVEL 4
How the Agent Thinks

Every time a customer texts, the bridge builds a prompt and pipes it to claude -p. Here's what the agent sees:

# The bridge builds this prompt for every message:

You are a WhatsApp business receptionist.
Reply to the customer's latest message.
Keep under 150 words. Warm, helpful, concise.

=== BUSINESS ===
(your BUSINESS.md: name, services, pricing, FAQ, brand voice)

=== CONVERSATION ===
Customer [10:32 AM]: Do you have dairy-free options?
Agent [10:32 AM]: Yes! We always have at least 2 sorbets...
Customer [10:33 AM]: Great, can I order for pickup?

BUSINESS.md = the knowledge

Everything the agent knows comes from this file. Services, pricing, hours, FAQ, brand voice. Edit it anytime.

conversations/ = the memory

Each customer gets a markdown file (by phone number). The bridge loads the last 30 lines as context. The agent remembers what was discussed.

Escalation = built-in

If the agent can't handle something (custom pricing, complaints, technical issues), it responds with ESCALATE: and the bridge logs it to escalations/ for you to review.

To change the agent's personality

Edit the Brand Voice section in BUSINESS.md. "Professional and direct" vs "Warm and playful" -- the agent adapts immediately. No retraining, no redeployment.

LEVEL 7
Run 24/7 with systemd

On a server (VPS), set up systemd so the bridge auto-starts on boot and restarts if it crashes.

Install the service

The repo includes a systemd unit file. Three commands to go from "running in terminal" to "runs forever."

sudo cp whatsapp-agent.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now whatsapp-agentcopy

Manage it

Standard systemd commands:

CommandWhat
systemctl status whatsapp-agentIs it running?
systemctl restart whatsapp-agentRestart it
journalctl -u whatsapp-agent -fWatch live logs
BUSINESS.md changes are instant. Edit the file and the next incoming message uses the new info. Only restart for bridge.mjs code changes.
