What We're Building

A WhatsApp number with an AI agent behind it that answers your customers 24/7.

Someone texts your business on WhatsApp. Instead of you replying manually, an AI agent reads the message, checks your business details, and responds in seconds. It knows your pricing, your hours, your FAQ. It handles orders, bookings, and basic questions. If something needs a human, it escalates to you.

Why build this?

You can't reply to every message yourself. Hiring someone costs $2,000+/mo. This costs under $120/mo and never sleeps. It's your receptionist, your first responder, your 24/7 front desk.

What you need

Access to Claude Code, a Zernio account for WhatsApp ($12/mo), a server to run it on ($4-10/mo), and 30 minutes to set it up.

What powers it

Claude Code's pipe mode (claude -p). You pipe a question in, get an answer out. One command. The bridge script does this for every incoming WhatsApp message automatically.

The WhatsApp Trigger catches incoming messages. The AI Agent (Claude Code in pipe mode) reads your business knowledge and conversation memory, then generates a reply. The Reply node sends it back to WhatsApp. If something needs a human, it goes to Escalations instead.

Example: Simple Question

INPUT Customer texts WhatsApp
"Do you guys deliver?"
CONTEXT Loads BUSINESS.md + conversation history
Delivery: within 5 miles, $5 fee, minimum $20
Hours: Mon-Thu 12-9, Fri-Sat 12-10, Sun 1-8
Previous messages: first contact, no history
THINK claude -p processes prompt with context
Reads business context + question, generates reply
OUTPUT Reply sent to WhatsApp
"Yes! We deliver within 5 miles of the shop. $5 delivery fee, $20 minimum order. Just send us what you'd like and your address!"

Example: Follow-up with Memory

INPUT Same customer, 2 minutes later
"Cool, can I get 2 scoops of mango and a chocovolcano?"
CONTEXT Loads BUSINESS.md + previous messages
Menu: Double scoop $6.50, Chocovolcano sundae $9
Mango Sorbet: available (dairy-free)
Previous: asked about delivery, told 5 miles/$5/$20 min
THINK Has full conversation context
Knows they want delivery. Calculates total. Asks for address.
OUTPUT
"Great choices! That's $6.50 for the double mango scoop + $9 for the Chocovolcano + $5 delivery = $20.50 total. What's your delivery address?"

Example: Escalation

INPUT
"I want to book your ice cream cart for a wedding, 200 guests, June 15. Can you do a custom menu?"
CONTEXT
Catering: starting $250, up to 30 guests, 3 flavors
200 guests = way beyond standard package
Custom menu = not in FAQ, needs owner
ESCALATE Beyond what the agent can handle
Reason: Custom catering for 200 guests, beyond standard package
Logged to: escalations/2026-03-30.md for you to review
OUTPUT Customer still gets a warm reply
"A wedding with 200 guests sounds amazing! That's a custom job so I want to make sure we get it perfect. Let me have Enrique reach out to you directly to plan the menu. He'll be in touch shortly!"
The "training" is just a markdown file. BUSINESS.md is everything the agent knows. Edit it and the next message uses the new info. No redeployment, no retraining, no waiting. Conversations are remembered per customer. Escalations get logged for you.

LEVEL 5
Run 24/7

On a VPS (private server), three commands make the agent run forever. Starts on boot. Restarts if it crashes.

VPS required. The bridge needs a server that's always on. A $4-10/mo VPS (Hetzner, DigitalOcean, etc.) works perfectly. This won't run 24/7 on your laptop. 
sudo cp whatsapp-agent.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now whatsapp-agentcopy

That's it. Check if it's running:

sudo systemctl status whatsapp-agentcopy

Update business info anytime

Edit BUSINESS.md. Changes take effect on the next incoming message. No restart needed. To update bridge.mjs code, run sudo systemctl restart whatsapp-agent.
